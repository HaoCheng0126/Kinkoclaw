## Design Context

### Users

Chinese-speaking macOS users who already run OpenClaw locally or on a remote host and want a lightweight desktop shell to access it. The product is used as a daily companion client: keep a character on the desktop, open the main stage quickly, chat through the existing gateway, and manage character presentation without touching the backend deployment.

### Brand Personality

Anime companion, precise, intimate. The interface should feel like a polished desktop character product rather than a generic admin console. It should stay technically clear while preserving a strong character-first presence.

### Aesthetic Direction

Reference the AIRI stage layout: model-first composition, left-side character stage and right-side conversation. The visual tone is soft dark plum with warm accent highlights, Chinese-first copy, and minimal chrome around the core interaction. Avoid dashboard clutter, duplicated entry points, or visible implementation/debug terminology in the main experience.

### Design Principles

1. Keep the character as the primary object; controls should stay secondary and collapse into a single settings entry.
2. Treat KinkoClaw as a thin client shell for OpenClaw, not as an operations console.
3. Support user-owned character libraries: importing and switching Live2D models must feel like a first-class workflow.
4. Prefer Chinese-first clarity for all primary stage copy, labels, and actions.
5. Never show duplicate character surfaces at the same time; desktop pet and stage should behave as one coherent presence.
